# 4주차 — CPU 스케줄링

## 🎯 학습 목표

- 스케줄링의 개념을 이해한다.
- 우선순위 스케줄링을 이해한다.
- CPU 스케줄링 알고리즘을 이해한다.

---

## 1. 스케줄링이란?

### 🍳 요리사 모형

식당 매니저가 예약/좌석/주문/조리 순서/손님 요청 등을 관리하듯, **운영체제는 CPU·메모리 자원을 효율적으로 배분하는 스케줄러**를 가집니다.

> **왜 필요한가?** CPU·메모리 자원은 **한정적**인데, 동작시킬 프로세스 개수는 매우 많기 때문.

---

## 2. 선점형 vs 비선점형 스케줄링

| 구분 | 선점형 (Preemptive) | 비선점형 (Non-Preemptive) |
|------|---------------------|---------------------------|
| **개념** | 실행 중인 프로세스의 CPU를 **강제로 뺏을 수 있음** | 한 번 CPU를 차지하면 **끝날 때까지 대기** |
| **반응성** | 빠름 | 느림 |
| **문맥교환** | 잦음 → 오버헤드 ↑ | 적음 → 효율 ↑ |
| **공평성** | 높음 | 낮음 |
| **대표 예시** | 시분할 시스템, 실시간 시스템 | 일괄 작업 시스템 |

---

## 3. 프로세스 우선순위

### 우선순위 계층 (높음 → 낮음)

```
커널 프로세스   ← 가장 높음 (운영체제 자체)
    ↓
전면 프로세스   ← 사용자와 상호작용, 빠른 응답 필요
    ↓
후면 프로세스   ← 백그라운드 동작
    ↓
입출력(I/O) 집중 프로세스   ← CPU burst 짧음
    ↓
CPU 집중 프로세스   ← CPU burst 김
```

### CPU 집중 vs I/O 집중

| 구분 | CPU 집중 | I/O 집중 |
|------|----------|----------|
| **CPU burst** | 길다 | 짧다 |
| **예시** | 복잡한 계산, 영상 인코딩 | 워드 프로세서, 입력 대기 |
| **우선순위** | 낮음 | **높음** |

> 💡 **핵심**: I/O 집중 프로세스를 먼저 처리하면 I/O 장치가 쉬지 않고 돌아가므로 **전체 시스템 효율 ↑**

### 전면/후면 프로세스

- **전면 프로세스**: GUI로 상호작용 중 → 빠른 응답 필요 → 우선순위 ↑
- **후면 프로세스**: 입력 없이 자동 동작 (백신, 업데이트 등)

---

## 4. 다중 큐 (Multiple Queue)

### 준비 상태 다중 큐

```
우선순위 1  →  [ P6 | P1 | P3 | P12 ] → CPU
우선순위 2  →  [ P9 | P11 | P5 ]
우선순위 3  →  [ P13 | P2 ]     ← 기아(아사) 발생 지점
```

우선순위에 따라 **여러 개의 큐**로 분리하여 관리. 높은 우선순위 큐부터 CPU 할당.

### 기아(Starvation, 아사) 현상

낮은 우선순위 프로세스가 계속 뒤로 밀려 **CPU를 영원히 받지 못하는 현상**.
> ⚠️ **스케줄러의 정책 문제** (교착상태와는 다름!)

### 고정 vs 변동 우선순위

| 방식 | 특징 |
|------|------|
| **고정 우선순위** | 생성 시 결정, 변하지 않음 — 단순, 기아 위험 |
| **변동 우선순위** | 실행 중 우선순위 조정 가능 — 복잡, 공정성 ↑ |

### 대기 상태 다중 큐

I/O 장치 **종류별로 분류**:

```
하드디스크  → [ P2, P7 ]
CD-ROM    → [ P4 ]
네트워크    → [ P8, P14 ]
```

**Interrupt Vector**에서 완료 신호를 받으면 해당 큐의 프로세스를 준비 상태로 이동.

## 5. CPU 스케줄링 평가 방법

평가 지표로 **평균 대기시간(Average Waiting Time)** 사용:

$$\text{AVR}(n) = \frac{\sum_{i=1}^{n} P_i}{n}$$

- $P_i$: i번째 프로세스의 대기시간
- 값이 **작을수록 좋은 알고리즘**

### 용어 정리

| 용어 | 정의 |
|------|------|
| **대기시간(Waiting Time)** | 준비 큐에서 기다린 시간 |
| **응답시간(Response Time)** | 첫 반응까지 걸린 시간 |
| **실행시간(Execution Time)** | 실제 CPU에서 처리된 시간 |
| **반환시간(Turnaround Time)** | 도착 ~ 종료 총 시간 |

## 6. 스케줄링 알고리즘 총정리

### 🔹 FCFS (First Come First Served)

- **비선점형** — 먼저 도착한 순서대로 처리
- ⭐ **단점**: **콘보이 효과(Convoy Effect)** — 앞의 긴 프로세스 때문에 뒤 짧은 프로세스들이 모두 대기

### 🔹 SJF (Shortest Job First)

- **비선점형** — 실행시간이 짧은 작업부터 처리
- ⭐ **장점**: 평균 대기시간 최소 (이론상 최적)
- ❌ **단점**: 실행시간 예측 어려움 + **기아(아사) 현상**

### 🔹 HRN (Highest Response Ratio Next)

- **비선점형** — SJF의 기아 문제 해결

$$\text{Priority}_i = \frac{W_i + \text{CPU}_i}{\text{CPU}_i}$$

- 대기시간이 길수록 우선순위 상승 → **에이징(Aging) 효과**

### 🔹 라운드 로빈 (Round Robin, RR)

- **선점형** — 모두가 **타임 슬라이스(Quantum)** 동안 순서대로 CPU 사용
- 가장 단순하고 공정
- ⚠️ 타임 슬라이스 설정이 관건
  - 너무 길면 → FCFS와 동일
  - 너무 짧으면 → 문맥교환 오버헤드

### 🔹 SRT (Shortest Remaining Time)

- **선점형** = SJF의 선점 버전 — 남은 실행시간이 가장 짧은 작업 선택
- 새 프로세스 도착 시마다 재평가 → 오버헤드 ↑
- 실무에서는 예측 어려워 드물게 사용

### 🔹 우선순위 스케줄링 (Priority Scheduling)

- **선점형 or 비선점형** 모두 가능
- **아사 현상 발생** — 에이징으로 해결

### 🔹 다단계 큐 (MLQ, Multi-Level Queue)

- 우선순위별로 **여러 큐**를 유지, 각 큐 간 고정 우선순위
- ❌ **단점**: 상위 큐에 프로세스가 계속 오면 하위 큐는 **기아**

### 🔹 다단계 피드백 큐 (MLFQ, Multi-Level Feedback Queue) ⭐

- **선점형** — MLQ의 기아 문제 해결
- 핵심 규칙:
  1. 실행을 마친 프로세스는 **한 단계 낮은 큐로 이동**
  2. 우선순위가 낮을수록 **타임 슬라이스를 길게** (이론상 최하위는 무한대)
- 유발 작업(CPU 집중) 처리에 효과적
- **현대 OS의 기본 스케줄러**

## 7. 알고리즘 비교표

| 알고리즘 | 선점 여부 | 선택 기준 | 기아 | 특징 |
|----------|---------|----------|------|------|
| **FCFS** | 비선점 | 도착 순서 | X | 단순, 콘보이 효과 |
| **SJF** | 비선점 | 최단 작업 | ⭕ | 평균 대기시간 최소 |
| **HRN** | 비선점 | 응답률 | X | SJF의 기아 해결 |
| **RR** | 선점 | 순환 (타임 슬라이스) | X | 공정, 타임슬라이스 민감 |
| **SRT** | 선점 | 최단 잔여 시간 | ⭕ | SJF의 선점 버전 |
| **우선순위** | 양쪽 | 우선순위 | ⭕ | Aging 필요 |
| **MLQ** | 양쪽 | 큐 우선순위 | ⭕ | 고정 할당 |
| **MLFQ** | 선점 | 동적 우선순위 | X | **현대 OS 기본** |

## 📐 평균 대기시간 계산 예제

**프로세스 도착**: P1(0, 10ms), P2(1, 4ms), P3(2, 6ms)

### FCFS
```
|── P1 ──|── P2 ──|── P3 ──|
0        10       14       20
```
- P1 대기=0, P2 대기=9(10-1), P3 대기=12(14-2)
- **평균 = (0+9+12)/3 = 7ms**

### SJF (비선점)
```
|── P1 ──|P2|── P3 ──|
0        10 14       20
```
- 동일 (첫 프로세스는 종료까지 안 뺏김)

### SRT (선점)
```
|P1|── P2 ──|P1|── P3 ──|── P1 ──|
0  1        5  6        12       20
```
- 새 프로세스 도착 시마다 남은 시간 비교

## 📝 핵심 요약

1. **스케줄링 = 자원을 효율적으로 배분하는 관리자 역할**
2. **선점형** 빠른 반응 ↔ **비선점형** 적은 오버헤드
3. 우선순위: **커널 > 전면 > 후면 > I/O 집중 > CPU 집중**
4. 다중 큐: 준비 상태는 우선순위별, 대기 상태는 장치 종류별
5. 평가 지표: **평균 대기시간**
6. **현대 OS 기본 = MLFQ** (다단계 피드백 큐)
